# Security Policy

## Supported Versions

Only the latest `V2.0` of **SphereWalk Campus Explorer** is fully supported with critical security patches.

| Version | Supported          |
| ------- | ------------------ |
| 2.0.x   | :white_check_mark: |
| < 2.0   | :x:                |

## Reporting a Vulnerability

Security is a primary priority for the SphereWalk system and the backend infrastructure. If you discover a security vulnerability within the platform, we appreciate your help in responsibly disclosing the issue to us securely.

**DO NOT** open a public issue on GitHub for security vulnerabilities.

Please send an email with the exact exploit details, Proof of Concept (PoC) code, and screenshots directly to the project author, **Lucky**.

- You will receive a response acknowledging receipt of your report within 24–48 hours.
- A patch will be deployed, and we will update you regarding the mitigation timeframe.
- If you desire, public credit for discovery will be granted once the fix is verified in production.

All code interactions must abide by the restrictive [LICENSE](LICENSE) terms at all times.
